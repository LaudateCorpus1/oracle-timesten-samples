Installing TimesTen Database on Linux and UNIX systems

This tutorial provides a step-by-step guide on how to install TimesTen databaes on Linux and UNIX systems.
Before installing TimesTen, ensure the appropriate requirements are met for your operating system.


1. Installing TimesTen Database on Linux and UNIX systems

To install TimesTen on your Linux and UNIX system, use the following steps:

  1. Download the TimesTen Database Distribution Medium from Oracle Software Delivery Cloud or OTN

    Note: In this tutorial,we will download the timesten221110.server.linux8664.tar.gz file under /tmp directory.

  2. Create TimesTen Installation Administrator and an Installation Directory

    Note: The user that installs TimesTen is the installation administrator. In this example, the installation administrator is timesten who owns the installation directory under /home/timesten/install directory.

  3. Extract the TimesTen Distribution Medium

    $ tar -zvxf /tmp/timesten221110.server.linux8664.tar.gz -C /home/timesten/install

    Note: On Linux and UNIX systems, after extracting the TimesTen Distribution Medium, an Installation is created. A TimesTen installation is read-only which means only the TimesTen installation administrator can make changes to an Installation.

2. Creating a TimesTen Instance

To create a TimesTen instance on your Linux and UNIX system, use the following steps:

  1. Instance Administrator

    Note: The user that creates TimesTen instance is the instance administrator. In this example, we will use the OS user timesten as both the installation and instance administrator who owns the Instance Home under /home/timesten/install/tt22.1.1.1.0 directory.

  2. Create an Instance

    $ cd /home/timesten/install/tt22.1.1.1.0/bin
    $ ./ttInstanceCreate

    You should follow the instructions and response to the prompt:

    Please choose an instance name for this installation? [ tt221 ] tt221

    Instance name will be 'tt221'.
    Is this correct? [ yes ] yes

    Where would you like to install the tt221 instance of TimesTen? [ /home/timesten ] /home/timesten/instance

    The directory /home/timesten/instance does not exist.
    Do you want to create it? [ yes ] yes

    The default port number is 6624.
    Do you want to use the default port number for the TimesTen daemon? [ yes ] yes

    Please enter a value for TNS_ADMIN (s=skip)? [ ] s

    What is the TCP/IP port number that you want the TimesTen Server to listen on? [ 6625 ] 6625

    Would you like to use TimesTen Replication with Oracle Clusterware? [ no ] no

    Note: Now, the TimesTen instance is created.

  3. Set the Instance Environment Variables

    $ source /home/timesten/instance/tt221/bin/ttenv.sh

    Note: The ttenv.* script will set TimesTen Instance environment including the Instance Home directory environment variable - TIMESTEN_HOME . If you want to change the configuration of the instance created, please execute ttInstanceModify command.

  4. Verify the Instance Status

    $ ttStatus

    Note: To confirm that the instance has been created correctly, check whether the database daemon and processes are running by entering the ttStatus command.
