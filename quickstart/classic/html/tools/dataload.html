Data Loading

The TimesTen Database provides many ways for data loading operations. This page contains how-to tutorials for some of these commonly used tools, utilities and built-in procedures.

  • ttSizeOra - ttSizeOra examines a table in an Oracle database and provides an estimate of the memory that would be required for the table, its metadata and its indexes if the table were to be.

  • ttBulkCp - The ttBulkCp utility enables you to copy a single table between databases, including between databases from different releases of TimesTen or between databases on different hardware platforms.

  • ttLoadFromOracle - This procedure takes a TimesTen table name, an Oracle SELECT statement and the number of threads for parallel load. It executes the query on the Oracle database and loads the result set into the specified TimesTen table.

  • ttImportFromOracle - This utility analyzes source tables in an Oracle Database and generates scripts to efficiently import them into a TimesTen database. It recommends space-efficient data types and identifies columns for columnar compression, generate DDL for the tables and indexes in the TimesTen database.
