How to use ttBulkCp

The tutorial shows how to use the ttBulkCp utility to export data from a TimesTen database and also how to import data from an external ASCII file into a TimesTen database table.

ttBulkCp is a TimesTen utility that can export from and import into TimesTen tables using ASCII files. It can be used to copy data between TimesTen databases. There are three majoe modes: output(-o) mode, input(-i) mode and the direct load input(-directLoad) mode.

The example on this page references a sample TimesTen database called sampledb. The database schema belongs to a user called ttuser with a sample table called EMPLOYEES.

1. Export data via output(-o) mode

In this section, you will learn how to export TimesTen database table via ttBulkCp utility output(-o) mode.

Examples:

The following command use ttBulkCp utility to export the ttuser.employees table from TimesTen using a text data file called employees.dump .

$ ttbulkcp -o -connstr "DSN=sampledb;uid=ttuser;pwd=ttuser" employees employees.dump

where

-o specifies the output mode to export data of a table from TimesTen database.
-oraconn specifies the connection string where DSN is the data source name, UID the user id and PWD the user password.
employees specifies the source table name to be copied in sampledb database.
employees.dump specifies the data file name.

 2. Import data via input(-i) mode

In this section, you will learn how to use the ttBulkCp utility input(-i) mode to import the data of single table into TimesTen database.

Examples:

  1. Before using ttBulkCp utility to load data into a TimesTen table, the table itself must first be created.

  2. The following command will load the rows listed in the file employees.dump into a table ttuser.employees in database sampledb .

    $ ttbulkcp -i "DSN=sampledb;uid=ttuser;pwd=ttuser" ttuser.employees employees.dump


3. Import data via direct load input(-directLoad) mode

In 18.1 release, there is a new major mode - direct load input(-directLoad) mode which is designed to load large amounts of data as fast as possible with the benefit of better performance and scalability.
Note: The direct load input mode of ttBulkCp utility is only available in Direct Connection Mode (ttBulkCp ), not Client/Server mode (ttBulkCpCS ).

In this section, we will use ttBulkCp with -directLoad option to load a large data file into TimesTen ttuser.employees table.

Examples:
  1. Connect to sampledb and verify the table ttuser.employees has been created.

  2. Load the rows listed in the file bigfile.dump into ttuser.employees table in database sampledb via -directLoad input mode .

    $ ttBulkCp -directload -connstr "DSN=sampledb;uid=ttuser;pwd=ttuser" ttuser.employees bigfile.dump


For more details and usage information, see ttBulkCp in the Oracle TimesTen In-Memory Database Reference.
